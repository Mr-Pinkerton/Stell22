Первый уровень 
Плитки “Основные показатели” (KPI) -
* Производство - кол-во произведенных изделий за период (данные с терминала - упаковка), % выполнения плана из раздела Цели. Под значением - изменение к предыдущему периоду.
* Поступления/Затраты - сумма поступлений и списаний за период (из Финансы - ДДС, операции типа “поступление” и “списание”). Под значением - изменение к предыдущему периоду.
* Остаток на счетах - сумма по всем счетам на текущую дату (Финансы). 


Второй уровень 
Блок “Требует внимания”. Система автоматически формирует список сигналов по данным из других разделов. Сверху показываются 3 самых срочных, остальные - в раскрывающемся списке “Все сигналы”. Каждый сигнал - цветной блок с заголовком, пояснением и ссылкой на нужный раздел. Возможные сигналы -
      * Материала хватит на N дней - рассчитывать по статистике последних 7 дней. 
      * Низкие остатки на складе - изделия, детали, упаковка или крепеж ниже минимального остатка. Установление остатков в настройках.
      * Высокий процент отхода - партия или работник превысил порог. Норма отхода задается в настройках.
      * Неразнесенные операции ДДС - кол-во строк банковской выписки без автоправила. Ссылка - Финансы - ДДС.
      * Расхождения инвентаризации - незакрытые отклонения по последней инвентаризации. Ссылка - Склад - Инвентаризация.
      * Цель под угрозой - если прогноз на конец периода ниже плана из раздела Цели. Ссылка - Цели.
Если сигналов нет - блок сворачивается в строку “Все в норме”.


Третий уровень
Графики (два в ряд) -
      * Производство vs план - столбчатый график. Ось X - недели выбранного периода. Ось Y - кол-во, шт. Две серии - факт и план. Линия плана на уровне недельной нормы из Целей.
      * Выручка vs прошлый месяц - линейный график. Ось X - недели периода. Ось Y - выручка, ₽ (только деньги, штуки на этой оси не смешиваются). Две серии - факт (₽) и прошлый месяц (₽).


Четвертый уровень 
Графики (два в ряд) -
      * Структура расходов - круговая диаграмма. Доли статей расходов за период в % (данные из Финансы - Статьи). Статьи - материал, зп производство, зп офис, аренда, электроэнергия, прочее (все статьи списаний из финансов, сгруппированные по корневым).
      * Процент отхода по дням - линейный график. Ось X - дни периода. Ось Y - отход, %. Горизонтальная линия порога 30%. Цвет точек - по той же логике, что в Отчеты - Процент отхода.

Прогрессбар - от зеленого к красному по мере уменьшения.


Общие правила расчета для дашборда -
      * Все периоды считаются по стандартным неделям.
      * Цвет KPI “Продажи” и “Производство” - линейная шкала: красный (0%) → желтый (~50%) → зеленый (100% и выше).
      * Сравнение с предыдущим периодом - для недели с прошлой неделей, для месяца с прошлым месяцем.

Правила учета материала
Правила учета материала
* Рейка является минимальной единицей учета материала.
* Пакет используется только для удобства хранения и выбора в терминале и не является отдельной учетной единицей.
* Один пакет может содержать рейки разных длин, сортов и типов.
* После подтверждения операции торцовки выбранные рейки считаются полностью израсходованными.
* Остаток рейки не возвращается на склад.
* Отход рассчитывается как разница между суммарной длиной выбранных реек и суммарной длиной произведенных деталей.

Для работников у кого указаны и ставка за час и расценки по операциям зарплата рассчитывается: cумма = (часы × ставка) + сдельная по операциям.
Для работников у кого не указаны расценки по операциям - он работает на окладе. Он вносит часы и операции один раз в сутки. Это считается сменой. Оплата идет только за часы, но по операциям мы считаем статистику - сумма за смену / кол-во операций = средняя стоимость операции (чаще всего это упаковка). Данные уходят в отчет по зп работника и в отчет по себестоимости операции.

Себестоимость 
Фильтры - Дата - Сбросить - Показать 
Табы - детали, изделия.
Детали - список произведенных за период деталей, с расценкой за работу, + себестоимостью материала. 
Расчет стоимости материала на деталь - вычисляется цена за метр доски с учетом сортности и процента отхода, и умножается на длину детали.
Если за период было несколько партий, то берется средняя стоимость по партиям с тем же типом и сортом.
Процент отхода — из Отчеты - Процент отхода партии.
Изделия -
Содержит таблицу с себестоимостью по каждому артикулу изделий за выбранный период. Расчет по умолчанию за текущий месяц, или по фильтрам. При смене периода пересчитываются все строки таблицы и доли статей.  Таблица -       
      * Название изделия/артикул
      * Материал (руб) (дополнительно указать долю в %)
      * Работа (руб) - торцовка, присадка, упаковка (дополнительно указать долю в %)
      * Прямая себестоимость (₽) — материал + работа + упаковка + крепеж + чекбоксы (дополнительно указать долю в %)
      * Накладные (производственные - категория в ддс) (₽) — доля постоянных расходов периода (дополнительно указать долю в %)
      * Полная себестоимость (₽) — прямая + накладные
При нажатии на строку — раскрывается детализация по деталям изделия: длина, кол-во в изделии, стоимость материала на деталь, расценки за операции.  
Состав себестоимости:       
      * Материал — рейки из Закупок с учетом процента отхода
      * Работа — сдельная или средняя из оклада зарплата производства (торцовка, присадка, упаковка)  
      * Производственные расходы
Расчет работы на изделие - По каждой детали в составе изделия (Номенклатура): расценка торцовки по сорту + расценка присадки, умножить на кол-во деталей в изделии. К сумме добавить расценку упаковки за изделие. Расценки — из карточки сотрудника (стандартные / средние по производству, не ставка конкретного работника).  
Прямая себестоимость изделия - сумма материала и работы по всем деталям + упаковка.  
Распределение накладных - Сумма постоянных расходов за период делится пропорционально прямой себестоимости всего произведенного за тот же период. 
Каждому изделию начисляется доля накладных = его прямая себестоимость * (накладные периода / сумма прямых затрат периода).

Правила расчета
      * Производственные операции оплачиваются по сдельным расценкам.
      * Работы, не связанные с производственными операциями (разгрузка, уборка, обслуживание оборудования и т.п.), оплачиваются по почасовой ставке.
      * В течение одного дня сотрудник может одновременно иметь сдельную и почасовую оплату.
      * Общая сумма зарплаты = сдельная часть + почасовая часть.

Терминал
Терминал — это сенсорный киоск, установленный на производстве. Его задача:
      1. Идентифицировать сотрудника (по PIN-коду).
      2. Давать доступ к операциям (Торцовка, Присадка, Упаковка, Рабочие часы).
      3. Фиксировать все производственные операции в реальном времени с привязкой к материалам, партиям, деталям и сотрудникам.
      4. Автоматически рассчитывать зарплату (сдельная + почасовая) и процент отхода.
      5. Обеспечивать контроль остатков материалов и готовой продукции.

Торцовка -
Первый блок - доступные партии - плитки с названиями, если их больше чем вмещается - горизонтальный скролл.
Второй блок - в первом блоке выбрана партия - во втором либо доступные рейки (если вносили поштучно) либо пакеты. При выборе пакета или реек плитки с партиями из первого блока становятся минимальными по высоте.
Третий блок - во втором блоке выбран пакет или рейки - плитки так же изменили высоту - появились табы “1 сорт” “2 сорт”. 
В табах все детали того типа, которого и рейка (канавка, полка).
Сорта могут быть любыми - из рейки 1 сорт могут быть детали 1 сорт и 2 сорт, из рейки 2 сорт могут быть детали 1 сорт и 2 сорт.
По тапу на плитку с деталью, появляется оверлей и модалка.
В ней заголовок с названием детали и подсказкой - введите количество произведенных деталей. Далее цифровая клавиатура.

Кнопки Отмена Подтвердить
Можно сделать множественный выбор деталей. 
Внизу кнопка подтвердить. 
Система отслеживает количество реек в пакете / партии, сотрудник не может взять больше чем есть. 
Система отслеживает количество и длину деталей, они не могут превышать длину того количества реек, которое взял сотрудник. Так же отслеживается процент отхода по партиям, пакетам (если есть) и сотрудникам.


Присадка -
Плитки всех произведенных деталей, можно выбрать несколько, такой же ввод количества как и в торцовке. 
Тот же функционал по внесению. 


Упаковка -
Плитки всех изделий внесенных в номенклатуру, на каждой плитке написано сколько  изделий можно собрать из уже произведенных деталей. 


Рабочие часы - можно внести часы, они рассчитаются по почасовой ставке.
После подтверждения упаковки система:
      * списывает использованные детали;
      * списывает упаковку и крепеж;
      * начисляет сдельную оплату упаковщику или вычисляет ее из оклада / кол-во изделий;
      * создает готовое изделие;
      * увеличивает остаток готовой продукции на складе.



Сценарий использования: работник торцовщик
      1. Вводит PIN-код → входит в терминал.
      2. Нажимает "Торцовка".
      3. Выбирает партию "Волочек 2419". Строка с партиями меняется по высоте (становится меньше), раскрывается строка с пакетами в партии. 
      4. Выбирает пакет №3 (рейки 2.4м, 50 шт), вводит количество взятых реек.
      5. Выбирает деталь "Полка 600" (1 сорт) - вводит 45 деталей.
      6. Выбирает деталь "Канавка 720" (2 сорт) - вводит 200 деталей
      7. Система рассчитывает брак в %: Сумма длин взятых реек - сумма длин внесенных деталей. 
      8. Нажимает "Подтвердить" → детали приходуются на склад.
      9. Выходит из системы сам или авто выход через 30 сек бездействия.

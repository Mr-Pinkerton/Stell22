# Запуск Stell22 на сервере (VPS + Docker) — пошагово

Инструкция для человека без опыта администрирования. Делайте строго по
порядку. Всё, что нужно печатать, показано в блоках — копируйте целиком.

Технические файлы уже подготовлены в проекте:
`Dockerfile`, `docker-compose.prod.yml`, `Caddyfile`, `docker-entrypoint.sh`,
`.env.deploy.example`. Менять их не нужно.

---

## Что понадобится (купить заранее)

1. **VPS** — виртуальный сервер, Ubuntu 22.04/24.04, минимум 2 ГБ RAM.
   (Timeweb, Selectel, Reg.ru, Yandex Cloud и т.п. — ~500–1000 ₽/мес.)
   После оплаты хостинг пришлёт: **IP-адрес**, **логин `root`** и **пароль**.
2. **Домен** — адрес сайта, например `stell22.ру` (~200–1500 ₽/год).

---

## Шаг 1. Направить домен на сервер

В панели, где куплен домен, создайте **A-запись**:
- имя: `@` (или нужный поддомен, напр. `app`);
- значение: **IP-адрес вашего сервера**.

Обновление адреса может занять от нескольких минут до пары часов.

---

## Шаг 2. Зайти на сервер

На своём компьютере откройте терминал (на Windows — «PowerShell») и введите,
подставив свой IP:

```bash
ssh root@ВАШ_IP_АДРЕС
```

Введите пароль от сервера (при вводе он не отображается — это нормально).

---

## Шаг 3. Установить Docker (один раз)

Скопируйте и выполните на сервере целиком:

```bash
curl -fsSL https://get.docker.com | sh
```

Проверьте, что установилось:

```bash
docker --version && docker compose version
```

Должны увидеть номера версий — значит всё хорошо.

---

## Шаг 4. Загрузить программу на сервер

```bash
git clone git@github.com:Mr-Pinkerton/Stell22.git
cd Stell22
```

Если сервер попросит ключ/доступ к GitHub — проще склонировать по HTTPS:

```bash
git clone https://github.com/Mr-Pinkerton/Stell22.git
cd Stell22
```

---

## Шаг 5. Задать секреты

Сгенерируйте два секретных значения (выполните дважды, сохраните оба):

```bash
openssl rand -base64 24   # это будет POSTGRES_PASSWORD
openssl rand -base64 32   # это будет SESSION_SECRET
```

Создайте файл настроек из шаблона и откройте его:

```bash
cp .env.deploy.example .env
nano .env
```

В открывшемся редакторе впишите:
- `DOMAIN` — ваш домен (без `http://`), напр. `stell22.ру`;
- `POSTGRES_PASSWORD` — первое сгенерированное значение;
- `SESSION_SECRET` — второе сгенерированное значение.

Сохранить и выйти в nano: **Ctrl+O**, **Enter**, затем **Ctrl+X**.

---

## Шаг 6. Запустить

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

Первый запуск идёт несколько минут (собирается программа). При старте
автоматически создаётся структура базы данных (миграции).

Посмотреть, что всё поднялось:

```bash
docker compose -f docker-compose.prod.yml ps
```

Логи приложения (если что-то не так):

```bash
docker compose -f docker-compose.prod.yml logs -f app
```

Через 1–2 минуты Caddy получит HTTPS-сертификат автоматически.

---

## Шаг 7. Задать пароль администратора

```bash
docker compose -f docker-compose.prod.yml exec app \
  npx tsx scripts/set-admin-password.ts "ВАШ_СТОЙКИЙ_ПАРОЛЬ"
```

Логин администратора: `admin@stell22.local`, пароль — тот, что задали.

---

## Шаг 8. Проверить

1. Откройте в браузере `https://ваш-домен` — должна открыться страница входа
   с замочком (HTTPS).
2. Войдите под администратором.
3. Проверьте разделы: дашборд, закупки, номенклатура, производство, отчёты,
   финансы.
4. Создайте тестовую партию → произведите → убедитесь, что себестоимость
   считается.

Служебная проверка «жив ли сервер»: `https://ваш-домен/api/health` —
должно вернуть `{"status":"ok","db":"up",...}`.

---

## Резервные копии базы (ОБЯЗАТЕЛЬНО)

Данные себестоимости — самое ценное. Сделайте копию так:

```bash
docker compose -f docker-compose.prod.yml exec db \
  pg_dump -U stell22 stell22 > backup_$(date +%F).sql
```

Настройте автоматический ежедневный бэкап (через `cron`) и **один раз
проверьте восстановление** из копии. Помогу с этим отдельно.

---

## Обновление программы (когда выйдут изменения)

```bash
cd Stell22
git pull
docker compose -f docker-compose.prod.yml up -d --build
```

Миграции применятся сами при старте. Данные не теряются.

---

## Полезные команды

| Задача | Команда |
|--------|---------|
| Остановить всё | `docker compose -f docker-compose.prod.yml down` |
| Запустить снова | `docker compose -f docker-compose.prod.yml up -d` |
| Логи приложения | `docker compose -f docker-compose.prod.yml logs -f app` |
| Перезапустить приложение | `docker compose -f docker-compose.prod.yml restart app` |

---

## Важно помнить

- Программа рассчитана на **один экземпляр** (расчёт себестоимости идёт через
  внутреннюю очередь). Не запускайте несколько копий на разных серверах.
- Файл `.env` с секретами **не коммитить** и никому не пересылать.
- Ozon/WB уже подключены (ключи задаются в разделе «Настройки → API» внутри
  программы). Банк и термопринтер — пока заглушки.

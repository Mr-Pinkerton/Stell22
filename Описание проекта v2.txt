ОПИСАНИЕ ПРОЕКТА (v2)

Версия 2. Переработана для устранения логических противоречий.
Главная цель проекта — точное и непротиворечивое понимание себестоимости изделия.

В конце документа (раздел «МОДЕЛЬ СЕБЕСТОИМОСТИ») собраны все ключевые правила и формулы расчёта в одном месте — это источник истины для расчётов.


=====================================================================
СТРАНИЦА 1 — АВТОРИЗАЦИЯ
=====================================================================
Форма входа — почта, пароль + восстановление пароля через почту.
Аккаунты есть только у администраторов (полный доступ).
«Работник» — это не аккаунт, а сущность для идентификации в терминале:
у работника нет логина/пароля, он входит в терминал по PIN-коду.
В терминал сначала заходит администратор (со своего аккаунта), далее
работники работают через выбор ФИО + подтверждение PIN.


=====================================================================
СТРАНИЦА 2 — АДМИН-ПАНЕЛЬ (ГЛАВНАЯ)
=====================================================================
Хедер:
- Слева — кнопка сворачивания сайдбара. Логотип размещён в шапке самого
  сайдбара (загрузка png/svg — в настройках); отдельного блока под логотип
  в хедере нет. Это осознанное решение, расхождением не считать.
- В центре — дата в стиле "Сегодня четверг 04.06.2026 г". Часовой пояс UTC+3.
- Справа — имя пользователя, колокольчик с уведомлениями, кнопка "Выйти".

Body:
- Слева сайдбар со ссылками. В самом низу сайдбара — кнопка "В терминал".
  Отдельной кнопки "На главную" нет: админка и есть главная.
- Справа основное рабочее поле (main).

В main:
- Верхний уровень — слева заголовок (повторяет название ссылки), справа
  кнопки Экспорт и Добавить. Если в категории экспорт/добавление не
  предусмотрены — кнопки не видно.
- Второй уровень — блок фильтров. На каждой странице выводятся только
  нужные фильтры, в той же последовательности, что в списке ниже (слева
  направо).
- Третий уровень — контент страницы.

Фильтры (общий набор):
- Поиск по названию (имени) — AJAX.
- Фильтр по дате — по месяцам и по периоду между двумя датами.
- Фильтр по неделям — вспомогательный к фильтру по месяцам, используется
  только в отчёте по зарплатам. Если выбран месяц, можно выбрать неделю.
  В список недель месяца попадают все недели, у которых пятница попадает
  в этот месяц. Неделя считается с пятницы по четверг, расчётный день —
  пятница. (Во всех остальных местах используется СТАНДАРТНАЯ неделя —
  см. раздел «Определение недели».)
- Чекбокс "Показать архив" — показывает неактивные записи (уволенные
  работники, выработанные партии, изделия и детали в архиве).
- Кнопки "Сбросить" и "Показать/Найти".

Страницы-ссылки:
Дашборд, Закупки, Сотрудники, Номенклатура, Отчёты, Финансы,
Производство, Продажи, Склад, Цели, Настройки.

ОПРЕДЕЛЕНИЕ НЕДЕЛИ (единое правило)
- Везде используется СТАНДАРТНАЯ неделя (пн–вс).
- Единственное исключение — отчёт "Зарплаты": неделя пт–чт, расчётный
  день — пятница.


=====================================================================
ДАШБОРД
=====================================================================
Заголовок — без кнопок Экспорт и Добавить (экспорт на дашборде не
предусмотрен).
Фильтры — переключатель периода вместо стандартного блока:
Текущий месяц / Текущая неделя (стандартная) / Произвольный период.
По умолчанию — текущий месяц. При смене периода пересчитываются все
плитки, графики и таблицы.

Первый уровень — Плитки KPI:
- Производство — кол-во произведённых изделий за период (данные с
  терминала — упаковка). Показывается факт (шт) и изменение к предыдущему
  периоду. ЕДИНЫЙ % выполнения плана здесь НЕ показывается (т.к. целей
  может быть несколько по разным изделиям) — проценты выводятся в блоке
  "Цели и прогноз" по каждой цели отдельно.
- Поступления — сумма поступлений за период (Финансы — ДДС, тип
  "поступление") + изменение к предыдущему периоду.
- Затраты — сумма списаний за период (Финансы — ДДС, тип "списание") +
  изменение к предыдущему периоду.
  (Поступления и Затраты — две отдельные плитки.)
- Остаток на счетах — сумма по всем счетам на текущую дату (Финансы).

Второй уровень — Блок "Требует внимания":
Система автоматически формирует сигналы по данным других разделов.
Сверху 3 самых срочных, остальные — в раскрывающемся списке "Все сигналы".
Каждый сигнал — цветной блок с заголовком, пояснением и ссылкой.
Возможные сигналы:
- Материала хватит на N дней — по статистике последних 7 дней.
- Низкие остатки на складе — изделия/детали/упаковка/крепёж ниже минимума
  (минимумы задаются в настройках).
- Высокий процент отхода — партия или работник превысил порог (норма
  отхода задаётся в настройках).
- Неразнесённые операции ДДС — кол-во строк выписки без автоправила.
  Ссылка — Финансы — ДДС.
- Расхождения инвентаризации — незакрытые отклонения по последней
  инвентаризации. Ссылка — Склад — Инвентаризация.
- Расхождение сортности партии — фактическая доля 2 сорта превысила
  заявленную более чем на 10%. Ссылка — Отчёты — Закупки.
- Расхождение стоимости партии — введённая стоимость не совпала с
  расчётной. Ссылка — Закупки.
- Цель под угрозой — прогноз на конец периода ниже плана. Ссылка — Цели.
Если сигналов нет — блок сворачивается в строку "Все в норме".

Третий уровень — Графики (два в ряд):
- Производство vs план — столбчатый график. Ось X — недели периода
  (стандартные). Ось Y — кол-во, шт. Две серии — факт и план. Линия плана
  для каждой недели = сумма дневных планов рабочих дней этой недели
  (НЕ константа: в неделях с разным числом рабочих дней план разный).
- Выручка vs прошлый месяц — линейный график. Ось X — недели периода.
  Ось Y — выручка, ₽. Две серии — факт (₽) и прошлый месяц (₽).
  (Только деньги; штуки на эту ось не смешиваются.)

Четвёртый уровень — Графики (два в ряд):
- Структура расходов — круговая диаграмма. Доли статей расходов за период
  в % (Финансы — Статьи, сгруппированы по корневым).
- Процент отхода по дням — линейный график. Ось X — дни периода. Ось Y —
  отход, %. Горизонтальная линия порога (по умолчанию 30%, настраивается).
  Цвет точек — по логике Отчёты — Процент отхода.

Пятый уровень — Блок "Цели и прогноз":
Для каждой активной цели:
- Название цели (изделие + кол-во).
- Прогресс-бар — произведено / план (шт и %).
- Прогноз — "успеваем" / "не успеваем" к концу периода.

Шестой уровень — два блока в ряд:
- Таблица "Топ работников" (за период, сортировка по объёму производства):
  Сотрудник (ФИО) / Произведено (кол-во деталей/изделий) / Отход (%) /
  ЗП к выплате (из Отчёты — Зарплаты).
- Список "Остаток материала по партиям" (активные партии из Закупки):
  прогресс-бар от зелёного к красному по мере уменьшения.

Общие правила дашборда:
- Все периоды — по стандартным неделям.
- Цвет KPI "Производство" — линейная шкала: красный (0%) → жёлтый (~50%)
  → зелёный (100% и выше).
- Сравнение с предыдущим периодом: неделя — с прошлой неделей, месяц — с
  прошлым месяцем.


=====================================================================
ЗАКУПКИ
=====================================================================
Заголовок — Экспорт — Добавить партию. В форме добавления есть таб
"Простая закупка" (крепёж/упаковка/прочее) — отдельной кнопки не нужно.
Фильтры — Поиск по названию — Показать архив — Сбросить — Найти.

Закупки бывают двух типов:
1. Партия рейки (основной тип, с пакетами/поштучно, расчётом объёма).
2. Простые товары (крепёж, упаковка, прочее) — без реек и пакетов,
   только: наименование (из Номенклатуры) / количество / цена за единицу.
   Используется для прихода крепежа/упаковки/разного на склад с ценой.

Таблица (партии рейки):
- Партия (название)
- Стоимость закупочная (по счёту)
- Стоимость общая (закупочная + доставка/разгрузка из Сделок; при
  создании = закупочной, дополняется при привязке к Сделке)
- Рейки (кол-во)
- Общая длина (сумма длин всех реек)
- Объём партии (м3)
- Статус (в работе / в архиве)
- Дата закупки
- Действия (изменить, списать остаток, удалить)

Списать остаток — остаток списывается в отход; списание попадает в
статистику процента отхода партии.
Удалить — безвозвратное. Возможно только если по партии нет никаких
движений материала и финансовых операций, ИЛИ она добавлена не более
24 часов назад.

ФОРМА «Добавить закупку (партия рейки)»
Заголовок: "Информация о закупке. Заполните поля, отмеченные *".

Блок "Основная информация":
- Название партии*
- Сечение рейки* — ширина и высота в мм (для расчёта объёма). Формат
  чисел: 12.5 мм или целые. ОДНО сечение на всю партию.

Блок "Добавить рейку" (выделен лёгким цветом). Внутри табы:
"Добавить пакетами" (по умолчанию) / "Добавить поштучно".

ВАЖНО (исправление): один пакет содержит рейки ТОЛЬКО ОДНОЙ длины, типа
и сорта. Это нужно для однозначного расчёта взятой длины в терминале.

Добавить пакетами — поля:
- Длина рейки (м)*
- Тип рейки* — полка / канавка
- Сорт* — Сорт 1 / Сорт 2
- Количество реек*
- Ряды и слои (необязательно) — если указаны оба, количество считается
  автоматически как ряды × слои (приоритет над ручным вводом).
Каждому пакету присваивается уникальный код, печатается на принтере
этикеток (размер наклеек — в настройках).

Добавить поштучно — поля:
- Длина рейки (м)*
- Тип рейки* — полка / канавка
- Сорт* — Сорт 1 / Сорт 2
- Количество*
Рейки с одинаковой длиной, типом и сортом суммируются — строка-
предупреждение "Рейки с одинаковой длиной, типом и сортом автоматически
суммируются".

Кнопка "Добавить" — disabled, пока не заполнены все поля.
Блок статистики по внесённым пакетам — общая длина, общий объём, кол-во
пакетов/реек, диаграмма по сорту в %.
Ниже — строки внесённых пакетов с иконкой "удалить", в каждой строке:
данные пакета + общая длина пакета + общий объём пакета.

Блок "Финансовые данные":
- Стоимость партии* — ИСТИНА (то, что реально заплатили). Всё считается
  за м3.
- Закупочная цена 1 сорт (₽/м3) — реальная цена поставщика за сорт 1.
- Закупочная цена 2 сорт (₽/м3) — реальная цена поставщика за сорт 2.
Цены 1/2 сорта — это реальные деньги поставщика; задают пропорцию
распределения стоимости между сортами (см. МОДЕЛЬ СЕБЕСТОИМОСТИ).
Проверка: если (Цена1×ОбъёмСорт1 + Цена2×ОбъёмСорт2) ≠ Стоимость партии —
показать уведомление, повесить значок на партию в списке и сигнал на
дашборд. Истиной всегда считается "Стоимость партии".

Уведомление: "Данные о доставке и сопутствующих расходах можно добавить
в Сделках."

Кнопка "Добавить поле" — можно добавить произвольное поле (выбрать тип и
название); оно появится в блоке и в экспорте, его можно заполнить.
Поле "Примечание". Кнопки "Отмена" и "Добавить партию".

ПРАВИЛА УЧЁТА МАТЕРИАЛА
- Рейка — минимальная единица учёта материала.
- Пакет используется только для удобства хранения и выбора в терминале,
  отдельной учётной единицей не является.
- Один пакет содержит рейки одной длины, типа и сорта.
- После подтверждения торцовки выбранные рейки считаются полностью
  израсходованными. Остаток рейки на склад не возвращается.
- Отход = разница между суммарной длиной взятых реек и суммарной длиной
  произведённых деталей.

СОТРУДНИКИ

Заголовок — Экспорт — Добавить сотрудника.
Фильтры — Поиск по имени — Показать архив — Сбросить — Найти.

Таблица:
- ФИО
- PIN-код (блюр, видно по клику) — 4 цифры, не хэшируется. Используется
  только как подтверждение при выборе ФИО в терминале, не является
  средством защиты данных или подтверждением финансовых операций.
- Статус (Активен / Архив)
- Действия (Изменить / В архив / Удалить)

ФОРМА «Добавить сотрудника»
Блок "Основная информация":
- ФИО*
- Дата рождения
Блок "Доступ":
- PIN-код
Блок "Оплата":
- Почасовая — ставка в час
- Сдельная — Торцовка 1 сорт (₽/деталь), Торцовка 2 сорт (₽/деталь),
  Присадка торцевая (₽/деталь), Присадка по плоскости (₽/деталь),
  Упаковка (₽/изделие).

Правила оплаты:
- Производственные операции оплачиваются по сдельным расценкам.
- Работы, не связанные с производством (разгрузка, уборка, обслуживание
  и т.п.), оплачиваются по почасовой ставке.
- В течение дня сотрудник может иметь и сдельную, и почасовую оплату.
- Общая ЗП = сдельная часть + почасовая часть.
- Если указаны и ставка в час, и сдельные расценки:
  ЗП = (часы × ставка) + сдельная по операциям.
- Если сдельные расценки НЕ указаны — работник на окладе (часовой):
  оплата только за часы. Операции он вносит для статистики. Для
  себестоимости операции берётся: сумма оклада за смену / кол-во операций
  (чаще всего — упаковка). Данные идут в отчёт по ЗП и в себестоимость.

Кнопки "Отмена", "Создать сотрудника".

НОМЕНКЛАТУРА

Заголовок — Экспорт — Добавить.
Табы — Изделия / Детали / Крепёж / Упаковка / Разное.

Для табов Крепёж / Упаковка / Разное в карточке задаётся ЦЕНА за единицу
(вводится вручную). Эта цена используется в прямой себестоимости изделия.
Приход остатков этих позиций — через Закупки (тип "простые товары") или
Сделки.

ИЗДЕЛИЯ — таблица:
Название / Артикул / Сорт (1/2) / Детали (кол-во штук) / Статус / Действия.

ФОРМА «Добавить изделие» (Заголовок: "Создание изделия"):
- Название
- Артикул (как в МП)
- Крепёж (выбор из списка крепежа + кол-во)
- Упаковка (выбор из списка упаковки)
- Чекбоксы из таба "Разное" — всё созданное в табе можно добавить;
  каждый чекбокс добавляет себестоимость.
- Сорт — выпадающий список
- Цена продажи (это цена реализации, НЕ себестоимость; себестоимость
  считается системой и здесь не вводится)

Блок "Детали в изделии":
Пока не выбран сорт — предупреждение "Сначала выберите сорт изделия —
доступны будут только детали того же сорта", выбор деталей заблокирован.
Добавить деталь:
- Выберите деталь — список деталей нужного сорта
- Количество
При добавлении появляется строка: наименование, количество (можно
менять), иконка "удалить".

ДЕТАЛИ — ФОРМА «Создание детали»:
- Название детали*
- Длина детали, м* (формат: 0.72м)
- Тип (полка / канавка)
- Присадка (торцевая / по плоскости) — у детали может быть несколько
  типов присадки. Если выбраны оба типа — деталь становится готовой
  только после выполнения ОБЕИХ присадок (см. Терминал — Присадка).
- Сорт (1 / 2)
Кнопки "Отмена", "Создать деталь".

ТЕРМИНАЛ
=====================================================================
Терминал — сенсорный киоск на производстве. Задачи:
1. Идентифицировать сотрудника (выбор ФИО + подтверждение PIN).
2. Давать доступ к операциям (Торцовка, Присадка, Упаковка, Рабочие часы).
3. Фиксировать операции в реальном времени с привязкой к материалам,
   партиям, деталям и сотрудникам.
4. Рассчитывать ЗП (сдельная + почасовая) и процент отхода.
5. Контролировать остатки материалов и готовой продукции.

Главный экран — плитки: Торцовка / Присадка / Упаковка / Рабочие часы.
Если у кого-то день рождения — поздравление, видно всем.

Хедер: справа ФИО работника и кнопка "Выйти" (если вход не выполнен —
"Войти"). Вход — выбор ФИО + ввод PIN. Смена работника выполняется через
"Выйти" → "Войти"; выпадающего списка работников в хедере нет (осознанное
решение). Бездействие более 30 сек — выход на главный экран.

--- ТОРЦОВКА ---
Первый блок — доступные партии (плитки; при переполнении — горизонтальный
скролл).
Второй блок — после выбора партии: доступные рейки (если вносили
поштучно) или пакеты. Плитки партий из первого блока становятся
минимальными по высоте.
Третий блок — после выбора пакета/реек: табы "1 сорт" / "2 сорт". В табах
все детали того же типа, что и рейка (канавка/полка). Сорта деталей могут
быть любыми (из рейки 1 сорта могут выйти детали и 1, и 2 сорта).
По тапу на деталь — оверлей и модалка: заголовок с названием детали,
подсказка "введите количество", цифровая клавиатура (123 / 456 / 789 /
С 0 <), кнопки "Отмена" / "Подтвердить". Можно выбрать несколько деталей.
Внизу — кнопка "Подтвердить".

Контроль (пакет = одна длина реек, поэтому расчёты однозначны):
- Система отслеживает кол-во реек в пакете/партии — нельзя взять больше,
  чем есть.
- Взятая длина = кол-во взятых реек × длина рейки пакета.
- Суммарная длина деталей не может превышать взятую длину.
- Отслеживается процент отхода по партиям, пакетам и сотрудникам.

--- ПРИСАДКА ---
Отдельные плитки на каждый ТИП присадки требуемой детали, например
"Полка 600 — торцевая" и "Полка 600 — по плоскости". Можно выбрать
несколько, ввод количества как в торцовке.
Деталь становится ГОТОВОЙ (доступной для упаковки) только когда выполнены
ВСЕ требуемые ей типы присадки. Система хранит состояние "сделано N из M
присадок".

--- УПАКОВКА ---
Плитки всех изделий из номенклатуры; на каждой написано, сколько изделий
можно собрать. Это число = МИНИМУМ из доступного количества по всем
компонентам: готовые детали + крепёж + упаковка.
После подтверждения упаковки система:
- списывает использованные детали;
- списывает упаковку и крепёж;
- начисляет сдельную оплату упаковщику или вычисляет её из оклада
  (оклад смены / кол-во операций);
- создаёт готовое изделие;
- увеличивает остаток готовой продукции на складе.

--- РАБОЧИЕ ЧАСЫ ---
Внесение часов; рассчитываются по почасовой ставке.

СЦЕНАРИЙ: работник-торцовщик
1. Выбирает ФИО, вводит PIN → входит в терминал.
2. Нажимает "Торцовка".
3. Выбирает партию "Волочек 2419". Строка партий уменьшается, раскрывается
   строка пакетов.
4. Выбирает пакет №3 (рейки 2.4м, 50 шт), вводит кол-во взятых реек.
5. Выбирает деталь "Полка 600" (1 сорт) — вводит 45 деталей.
6. Выбирает деталь "Канавка 720" (2 сорт) — вводит 200 деталей.
7. Система считает отход: сумма длин взятых реек − сумма длин деталей.
8. "Подтвердить" → детали приходуются на склад.
9. Выход вручную или авто через 30 сек.

СИСТЕМА УВЕДОМЛЕНИЙ
В терминале уведомления приходят справа под хедером, друг под другом, не
перекрываясь. Ошибка — красный, Успех — зелёный, Информирование — синий.
В админ-панели дополнительно — колокольчик с цифрой, по клику окно с
уведомлениями.


=====================================================================
МОДЕЛЬ СЕБЕСТОИМОСТИ (источник истины для расчётов)
=====================================================================

ОБЩИЕ ПРИНЦИПЫ
- Учёт материала ведётся строго по партии-источнику: терминал при торцовке
  знает конкретную партию и пакет, поэтому каждая деталь жёстко привязана
  к своей партии. Усреднение по периоду НЕ используется.
- Все расчёты материала ведутся в м3 внутри партии (сечение одно на
  партию, поэтому "погонный метр ↔ м3" внутри партии — константа).
- Себестоимость фиксируется снапшотом на момент операции (предварительно)
  и замораживается при закрытии партии/месяца ПОСЛЕ выплаты ЗП по всем её
  операциям (итог). До выплаты правка операции пересчитывает предварительный
  снапшот (правка после выплаты уже невозможна).

1. МАТЕРИАЛ
Дано по партии:
  C        — Стоимость партии (истина: закупочная + доставка из Сделок).
  P1, P2   — цены поставщика за м3 по сорту 1 и сорту 2 (реальные деньги).
  Сечение  — одно на партию.
Факт из производства (по произведённым деталям этой партии):
  V1ф = (сумма длин деталей 1 сорта) × Сечение  — фактический объём 1 сорта
  V2ф = (сумма длин деталей 2 сорта) × Сечение  — фактический объём 2 сорта

Распределение стоимости по сортам (вся уплаченная сумма C ложится на
фактически произведённые детали; отход и потери при этом полностью зашиты
в себестоимость, сумма сходится к C):

  Доля_1 = (P1 × V1ф) / (P1 × V1ф + P2 × V2ф)
  Доля_2 = (P2 × V2ф) / (P1 × V1ф + P2 × V2ф)

  Стоимость сорта 1 = C × Доля_1
  Стоимость сорта 2 = C × Доля_2

  Цена за м3 (сорт 1, с учётом отхода) = (C × Доля_1) / V1ф
  Цена за м3 (сорт 2, с учётом отхода) = (C × Доля_2) / V2ф

  Материал на деталь = Цена за м3 (по сорту детали) × Длина детали × Сечение

Аналитика: сравниваем "Соотношение сортов закупка" (заявленное) и
"Соотношение сортов факт"; при превышении факт. доли 2 сорта более чем на
10% — сигнал на дашборд.

Фиксация: пока партия "в работе" — V1ф/V2ф накапливаются, себестоимость
ПРЕДВАРИТЕЛЬНАЯ. Закрытие (выработана/архив, доставка из Сделок учтена) само
по себе НЕ замораживает: снапшот остаётся предварительным и пересчитывается
при правках операций до выплаты. ЗАМОРОЗКА (окончательный пересчёт, FINAL,
поле frozenAt) наступает, когда партия закрыта И выплачена ЗП по всем её
операциям торцовки — после этого распределение править нельзя.

Два уровня учёта материала (важно):
1) Уровень заготовки/детали — СТРОГО по партии-источнику. Терминал при
   торцовке знает партию, поэтому себестоимость каждой произведённой
   заготовки/детали и анализ отхода считаются по её партии, без усреднения.
2) Уровень изделия (артикул) за период — по средневзвешенной цене породы
   (WAC), а НЕ по конкретной партии. Причина: после торцовки заготовки одной
   породы/длины/типа/сорта попадают в общий склад заготовок (BlankStock) и
   физически обезличиваются — партия по слоям не отслеживается. Поэтому:

     ₽/м (порода, сорт) = Σ Стоимость_сорта по партиям этой породы
                        / Σ произведённая длина_сорта по партиям этой породы
     Материал детали изделия = ₽/м(порода, сорт детали) × длина детали

   Усреднение ограничено ОДНОЙ породой (разные породы не смешиваются). Вся
   стоимость партий породы распределяется на произведённое → суммы сходятся.
   Точная привязка «партия → изделие» потребовала бы учёта слоёв FIFO на
   складе заготовок; при стабильных ценах партий WAC даёт ту же величину.

2. РАБОТА
Расчёт работы на изделие — по каждой детали состава:
  расценка торцовки (по сорту детали) + расценки требуемых присадок
  (торцевая и/или по плоскости), умноженные на кол-во деталей в изделии,
  + расценка упаковки за изделие.
Источник расценок:
  - Сдельщики — по фактическим расценкам работника, выполнившего операцию.
  - Окладники (например, упаковщики) — оплата по часам; в себестоимость
    операции берётся (оклад смены / кол-во операций).
  - Для строки "Себестоимость по артикулу" за период берётся фактическая
    средняя по производству для каждой операции (смесь сдельных и
    окладных).
ЗП производства НЕ входит в накладные (см. Финансы — Статьи).

3. КРЕПЁЖ / УПАКОВКА / РАЗНОЕ
Цена за единицу — из карточки Номенклатуры (вводится вручную).
В себестоимость изделия добавляется: цена крепежа × кол-во + цена упаковки
+ суммы выбранных чекбоксов "Разное".

4. ПРЯМАЯ СЕБЕСТОИМОСТЬ ИЗДЕЛИЯ
  Прямая = материал по всем деталям + работа по всем деталям + упаковка
           (работа) + крепёж + упаковка (материал) + чекбоксы.

5. НАКЛАДНЫЕ (производственные расходы — категория в ДДС)
Сумма постоянных производственных расходов периода распределяется
пропорционально прямой себестоимости всего произведённого за тот же период:
  Накладные изделия = Прямая изделия × (Накладные периода / Сумма прямых
                      затрат периода)
Timing: в середине месяца — предварительно по накопленным на дату
расходам; окончательный пересчёт при закрытии месяца.

6. ПОЛНАЯ СЕБЕСТОИМОСТЬ
  Полная = Прямая + Накладные.

7. НДС И ОКРУГЛЕНИЕ
НДС в себестоимости не учитывается (упрощёнка): все суммы берутся как в
выписке/счёте. Округление денег для расчёта купюр — до 100 по правилам
математики (50 и более — вверх).

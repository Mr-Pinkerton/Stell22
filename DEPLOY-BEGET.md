# Запуск Stell22 на Beget (VPS + Docker) — пошагово

Инструкция для человека без опыта администрирования. Делайте по порядку,
команды копируйте целиком. Технические файлы (`Dockerfile`,
`docker-compose.prod.yml`, `Caddyfile` и др.) уже готовы в проекте.

---

## Что понадобится

- Аккаунт на [beget.com](https://beget.com) с пополненным балансом.
- Домен (можно купить прямо в Beget).

---

## Шаг 1. Заказать VPS

1. Войдите в панель Beget → раздел **«VPS»** (Облачные серверы).
2. Нажмите создать сервер, выберите:
   - ОС: **Ubuntu 24.04**;
   - тариф: минимум **2 ГБ RAM** (для запаса — 4 ГБ).
3. Создайте сервер. Через ~1 минуту в панели появятся:
   - **IP-адрес** сервера;
   - **пароль root** (или он придёт на почту).

> У Beget есть бесплатное резервное копирование самого сервера — включите его
> в настройках VPS. Это дополнительная страховка помимо бэкапов базы (шаг 8).

---

## Шаг 2. Направить домен на сервер

В панели Beget → раздел **«Домены»** или **«DNS»**:

1. Нажмите **«Добавить запись»**.
2. Тип: **A**.
3. Имя: `@` (для основного домена; или, например, `app` для поддомена).
4. Значение: **IP-адрес вашего VPS**.
5. Сохраните.

Обновление адреса в интернете занимает от нескольких минут до пары часов.

---

## Шаг 3. Подключиться к серверу

На своём компьютере откройте терминал (Windows — «PowerShell») и введите,
подставив свой IP:

```bash
ssh root@ВАШ_IP
```

Введите пароль (при вводе он не виден — это нормально). При первом входе
согласитесь с вопросом про отпечаток (наберите `yes`).

---

## Шаг 4. Базовая защита сервера (рекомендуется)

Обновите систему и включите файрвол — пропускаем только нужные порты:

```bash
apt update && apt upgrade -y
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
```

---

## Шаг 5. Установить Docker (один раз)

```bash
curl -fsSL https://get.docker.com | sh
docker --version && docker compose version
```

Должны увидеть номера версий.

---

## Шаг 6. Загрузить программу

```bash
git clone https://github.com/Mr-Pinkerton/Stell22.git
cd Stell22
```

---

## Шаг 7. Задать секреты и запустить

Сгенерируйте два секрета (выполните по очереди, сохраните оба):

```bash
openssl rand -base64 24   # → это POSTGRES_PASSWORD
openssl rand -base64 32   # → это SESSION_SECRET
```

Создайте файл настроек и откройте его:

```bash
cp .env.deploy.example .env
nano .env
```

Заполните:
- `DOMAIN` — ваш домен без `http://` (напр. `stell22.ру`);
- `POSTGRES_PASSWORD` — первый секрет;
- `SESSION_SECRET` — второй секрет;
- `SENTRY_DSN` — оставьте пустым (подключим позже, шаг 9).

Сохранить в nano: **Ctrl+O**, **Enter**, выйти: **Ctrl+X**.

Запустите (первый раз идёт несколько минут — собирается программа):

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

Проверьте статус и логи:

```bash
docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml logs -f app
```

Через 1–2 минуты Caddy автоматически получит HTTPS-сертификат.

---

## Шаг 8. Пароль администратора

```bash
docker compose -f docker-compose.prod.yml exec app \
  npx tsx scripts/set-admin-password.ts "ВАШ_СТОЙКИЙ_ПАРОЛЬ"
```

Вход: `admin@stell22.local` + заданный пароль.

Откройте `https://ваш-домен` — должна открыться страница входа с замочком.

---

## Шаг 9. Автоматические бэкапы базы (ВАЖНО)

В проекте есть готовые скрипты `scripts/backup-db.sh` и `scripts/restore-db.sh`.

**Проверьте, что бэкап делается:**

```bash
sh scripts/backup-db.sh
```

Появится файл в папке `backups/`.

**Настройте ежедневный автобэкап** в 03:00. Откройте расписание:

```bash
crontab -e
```

Добавьте строку (путь — там, куда склонировали проект; обычно `/root/Stell22`):

```
0 3 * * * cd /root/Stell22 && sh scripts/backup-db.sh >> /root/Stell22/backups/backup.log 2>&1
```

Сохраните и выйдите. Копии хранятся 14 дней (старые удаляются сами).

**Проверьте восстановление** (на тестовом окружении, до реального старта):

```bash
sh scripts/restore-db.sh backups/ИМЯ_ФАЙЛА.sql.gz
```

## Шаг 10. Мониторинг ошибок Sentry (необязательно)

Если хотите получать уведомления о сбоях:

1. Зарегистрируйтесь на [sentry.io](https://sentry.io), создайте проект
   типа **Next.js**.
2. Скопируйте **DSN** (Settings → Projects → Client Keys).
3. На сервере впишите его в `.env`:
   ```bash
   nano .env      # SENTRY_DSN=ваш_dsn
   ```
4. Перезапустите приложение:
   ```bash
   docker compose -f docker-compose.prod.yml up -d
   ```

Серверные ошибки начнут приходить в Sentry. (Ошибки в браузере — отдельная
опция `NEXT_PUBLIC_SENTRY_DSN`, требует пересборки; подключим при желании.)

## Проверка после запуска

- `https://ваш-домен` открывается по HTTPS, вход работает.
- Разделы грузятся: дашборд, закупки, номенклатура, производство, отчёты,
  финансы.
- Тестовая партия → производство → себестоимость считается.
- Служебная проверка: `https://ваш-домен/api/health` → `{"status":"ok",...}`.

## Обновление программы

```bash
cd /root/Stell22
git pull
docker compose -f docker-compose.prod.yml up -d --build
```

Миграции применяются автоматически, данные сохраняются.

---

## Полезные команды

| Задача | Команда |
|--------|---------|
| Статус | `docker compose -f docker-compose.prod.yml ps` |
| Логи приложения | `docker compose -f docker-compose.prod.yml logs -f app` |
| Перезапуск | `docker compose -f docker-compose.prod.yml restart app` |
| Остановить всё | `docker compose -f docker-compose.prod.yml down` |
| Бэкап вручную | `sh scripts/backup-db.sh` |

---

## Важно помнить

- Программа рассчитана на **один экземпляр** — не запускайте копии на разных
  серверах (расчёт себестоимости идёт через внутреннюю очередь).
- Файл `.env` с секретами **не коммитить** и никому не передавать.
- Ozon/WB подключаются в самой программе: «Настройки → API». Банк и
  термопринтер пока заглушки.
